# Playtest 2026-05-13 — bugs found by driving Unity via MCP

User reported: "tôi thấy sai hết à". Discovered by running `mcp__unity-mcp__manage_editor play` + `execute_code` to invoke interactions + read state. All fixed in commits `a6a3757` and `b1e5e11`.

## B-001 — BootstrapEntry did not load any world scene
- **Symptom**: Press Play in 00_Bootstrap → services initialized, console shows "[ServiceLocator] Bootstrap done", but game window only shows the UI Canvas. No Player, no NPCs, no terrain.
- **Cause**: `BootstrapEntry.Awake` called `services.Bootstrap() + Quests.StartDay(1)` but never loaded any other scene. User has nothing to interact with.
- **Fix**: Added `[SerializeField] SceneRefSO firstScene` to BootstrapEntry, defaults to `SceneRef_01_MainMenu`. SceneBuilder wires this. After Bootstrap → `Scenes.LoadSingle(firstScene)` → MainMenu shows.
- **Tracked in**: [[systems/scene-flow]].

## B-002 — InteractionSO concretes were no-op stubs
- **Symptom**: Player walks to SanVanDong, presses E → only `Debug.Log("[Confirm] ...")` fires. No popup, no time skip, no quest complete. Quest then naturally MISSED at deadline → -5 renLuyen.
- **Cause**: `OpenConfirmInteractionSO.Execute`, `SceneTransitionInteractionSO.Execute`, etc. were all stubs from earlier development. `InteractionContext` had only `player` + `area` — no service hooks injected.
- **Fix**: `InteractionContext` extended with `Func`/`Action` delegates (`showConfirm`, `loadAdditive`, `skipTimeTo`, `completeCurrentQuest`, `showDialogue`, `showQuiz`, `applyScoreDelta`, `showLoading`, `advanceDay`). `InteractionRouter.HandlePressed` populates context with delegates routed to `IUIRouter`/`ISceneRouter`/`IGameClock`/`IScoreSystem`. SO.Base stays Services-free (no direct ref) via the delegate pattern.
  - `OpenConfirmInteractionSO`: showConfirm → if OK → skipTimeTo(h,m) → completeCurrentQuest(true).
  - `SceneTransitionInteractionSO`: loadAdditive(targetScene) → completeCurrentQuest(true).
  - `OpenQuizInteractionSO`: showQuiz → compute score from correct/total*maxPerLesson → applyScoreDelta → complete.
  - New `SleepInteractionSO`: showConfirm("Di ngu?") → showLoading 2s → completeCurrentQuest(true) → advanceDay().

## B-003 — Double penalty on quest miss (-10 instead of -5)
- **Symptom**: User reports renLuyen drops by 10 per missed quest. GDD says 5.
- **Cause**: `QuestRouter.MissCurrent` called `_scoreSystem.ApplyDelta(0, -q.latePenalty, q.id)` AND `_activeQuest.runtimeInstance.OnComplete(false)` which in `GotoConfirmQuestRuntime` calls `_ctx.awardScoreDelta(-_so.latePenalty)`. Both eventually hit `ScoreSystem.ApplyDelta` → -5 + -5 = -10.
- **Fix**: Removed direct ApplyDelta call from `QuestRouter.MissCurrent`. Runtime's `OnComplete(false)` is now the single source of penalty. Each quest type can decide its own penalty (e.g. FreeRoam = 0).
- **Verified**: 3 missed quests → renLuyen 100 → 85 (-15 = 3 × 5) ✓.

## B-004 — IUIRouter stub kept firing after UIRouterMono.OverrideUI
- **Symptom**: UIConfirm.ShowAsync never called the real Mono. Only "[UIRouter stub] ShowConfirm" appeared.
- **Cause**: Boot order:
  1. BootstrapEntry (`-1000`) → ServiceLocator.Bootstrap → `UI = new UIRouter() /* stub */`. Services like InteractionRouter cache `_ui` = stub.
  2. UIRouterMono.Awake (default 0) → `services.OverrideUI(this)` → swaps `ServiceLocator.UI` field to point at Mono.
  3. But InteractionRouter._ui still points at the stub it cached at step 1.
- **Fix**: New `UIRouterFacade : IUIRouter`. ServiceLocator holds the facade with `Inner = stub` initially. `OverrideUI` swaps `facade.Inner = mono` instead of replacing the field. All cached `_ui` refs now route via facade to current backend.
- **Tracked in**: [[decisions/d13-ui-router-facade]].

## B-005 — Sleep quest had no day-rollover trigger
- **Symptom**: At 18:30, Sleep quest active. Press E at KTX_Door → loaded `13_KyTucXa` sub-scene but day never advanced.
- **Cause**: `SleepQuestSO.OnComplete` was empty. `KTX_Door` interactable was `SceneTransitionInteractionSO` (scene load only, no day logic). No actor was responsible for clock advancement on sleep.
- **Fix**: New `SleepInteractionSO` does the rollover orchestration: showConfirm → showLoading → completeCurrentQuest(true) → advanceDay (which calls `Clock.AdvanceDay()` + `Clock.SkipTo(4,30)`). `QuestRouter` now subscribes to `DayStartedMsg` and auto-calls `StartDay(msg.day)` so the new day's quests get queued. SOGenerator maps `KTX_Door` interactable to `SleepInteractionSO` (skip the dorm sub-scene step for v1).
- **Verified**: 18:30 → interact KTX → confirm popup "Di ngu?" → click OK → day=2, clock=04:34, weekday=2, Quest_02_01_Tap_the_duc auto-active.

## B-006 — Input keys did nothing (Player not movable)
- **Symptom**: User: "Game đang bị lỗi hệ thống input thế chơi như nào?"
- **Cause**: PlayerController, ThirdPersonCameraRig, PlayerInteractor used `[SerializeField] InputActionReference` fields. No `.inputactions` asset exists in project → refs never wired → keys silently ignored.
- **Fix**: Rewrite to use `UnityEngine.InputSystem.Keyboard.current` / `Mouse.current` / `Gamepad.current` directly. No SerializeField refs needed. WASD + arrows + gamepad stick for move. RMB-drag mouse for camera orbit (Mouse.delta). `Keyboard.current[Key.E].wasPressedThisFrame` for interact.
- **Tracked in**: [[decisions/d12-direct-input-api]].
